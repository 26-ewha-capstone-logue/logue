# Logue PMF 정의 및 경쟁사 비교

## 1. Logue의 PMF (Product-Market Fit)

Logue는 BI 이후 단계에서 발생하는 ‘후속 질문 구조화와 해석 병목’을 해결하는 서비스다. 기존 BI 도구가 데이터를 보여주는 데 집중하는 반면, Logue는 KPI 변화 이후 반드시 발생하는 “왜?”라는 질문을 분석 가능한 구조로 변환하고, 해석 가능한 결과로 연결하는 데 초점을 둔다.

특히 마케팅·CRM·그로스 실무자는 KPI 변동 이후 반복적으로 원인 설명을 요구받지만, 분석 자체는 주 업무가 아니기 때문에 질문 정의, 지표 기준 설정, 해석 과정에서 병목이 발생한다.

Logue의 PMF는 다음과 같이 정의된다.

- 자연어 질문 → 분석 구조로 변환
- 지표/기간/기준/필터 자동 정렬
- 결과 해석까지 포함한 일관된 분석 제공
- 실무자가 “직접 설명할 수 있는 상태”까지 도달

즉, Logue는  
“데이터를 보는 도구가 아니라, 분석 질문과 해석을 완성시키는 도구”다.

## 2. 경쟁사 선정

- ThoughtSpot → NLQ 기반 검색형 BI
- Tableau → 시각화 중심 BI
- Looker → semantic layer 기반 BI

## 4. 핵심 차이

기존 BI 시장은 크게 두 방향으로 나뉜다.

Tableau와 같은 도구는 데이터 시각화와 리포팅에 강점을 가지며, 사용자가 데이터를 직접 탐색하도록 돕는다.

ThoughtSpot과 같은 도구는 자연어 기반 검색을 통해 질문 진입 장벽을 낮추는 방향으로 진화하고 있다.

Looker는 semantic layer를 통해 지표 정의의 일관성과 데이터 거버넌스를 강화하는 데 집중한다.

하지만 이들 모두 공통적으로 다음 한계를 가진다.

- 질문을 받는 것까지는 가능하지만, 그 질문을 분석 구조로 정리하는 과정은 사용자에게 남아 있음
- 결과를 보여주지만, 실무자가 직접 설명 가능한 수준까지 해석을 완성하지는 않음

Logue는 이 지점에서 다르다.

- NLQ → SQL 변환이 아니라, NLQ → 분석 설계(analysis spec)로 변환
- 단순 결과 제공이 아니라, “왜 이런 결과가 나왔는지 설명 가능한 상태”까지 지원
- BI 이전/이후가 아니라, BI 이후 ‘해석 단계’를 대체

## 5. 한 줄 정리

- Tableau / Looker / ThoughtSpot → 데이터를 보여주거나 찾게 해주는 도구
- Logue → 데이터를 이해하고 설명하게 만드는 도구
